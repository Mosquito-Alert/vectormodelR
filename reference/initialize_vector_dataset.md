# Build a unified vector presence dataset

Combines previously prepared Mosquito Alert and/or GBIF vector
observations for a given location into a single presence-only dataset
with harmonised columns. The helper looks for the size-specific vector
files generated by
[`get_malert_data()`](https://labs.mosquitoalert.com/mosquitoR/reference/get_malert_data.md)
and
[`get_gbif_data()`](https://labs.mosquitoalert.com/mosquitoR/reference/get_gbif_data.md)
and merges them when requested.

## Usage

``` r
initialize_vector_dataset(
  iso3,
  admin_level,
  admin_name,
  sources = c("malert", "gbif"),
  vector_dirs = "data/proc",
  output_dir = "data/proc",
  overwrite = FALSE,
  exclude_collection_on_merge = "Mosquito Alert Tiger Mosquito"
)
```

## Arguments

- iso3:

  Three-letter ISO3 country code used in the vector filenames.

- admin_level:

  Administrative level included in the vector filenames.

- admin_name:

  Administrative unit name used in the vector filenames.

- sources:

  Character vector specifying which vector sources to include. Accepted
  values are `"malert"`, `"gbif"`, or both. Default includes both.

- vector_dirs:

  Character vector of directories to search for the prepared vector
  files. Defaults to `"data/proc"`.

- output_dir:

  Directory where the combined dataset will be saved. Defaults to
  `"data/proc"`.

- overwrite:

  Logical; set to `TRUE` to overwrite an existing output file. Defaults
  to `FALSE`.

- exclude_collection_on_merge:

  Character vector of GBIF `collectionCode` values to drop when both
  sources are merged. Defaults to discarding
  `"Mosquito Alert Tiger Mosquito"` when Mosquito Alert data is present.
  Use `NULL` to keep all collection codes.

## Value

A tibble with columns `date`, `year`, `lon`, `lat`, `presence`,
`referenceID`, and `source`. Attributes include `output_path` and
`source_files`.
